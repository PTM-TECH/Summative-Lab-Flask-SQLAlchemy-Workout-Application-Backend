# Summative-Lab-Flask-SQLAlchemy-Workout-Application-Backend

# Project Title
## Workout Tracker API

# Project Description
The Workout Tracker API is a Flask-based backend system designed to help personal trainers and fitness users track workouts and exercises.

The API allows users to:
- Create, view, and delete workouts
- Create, view, and delete exercises
- Associate exercises with workouts, including performance data such as reps, sets, and duration

The system is built using:
- Flask (REST API framework)
- SQLAlchemy (ORM for database modeling)
- Flask-Migrate (database migrations)
- Marshmallow (serialization and validation)

It follows RESTful design principles and supports relational data through a many-to-many relationship between workouts and exercises.


## Installation Instructions

### 1. Clone the repository
- git clone git@github.com:PTM-TECH/Summative-Lab-Flask-SQLAlchemy-Workout-Application-Backend.git
- open in vscode -> then run
- cd server
### 2. Create Virtual Environment and install dependencies
- pipenv install
- pipenv shell
- pip install -r requirements.txt
### 3. Set up the database
- flask db init
- flask db migrate -m "initial migration"
- flask db upgrade
### 4. Seed the database
- python3 seed.py

## Run Instructions
- Start the flask server
    - python3 app.py
- http://127.0.0.1:5555

## API Endpoints

### Workouts
GET/workouts
- Returns all workouts
GET/workouts/<int:id>
- Returns a single workout including associated exercises.
POST/workouts
- Creates a new workout
DELETE/workouts/<int:id>
- Deletes a workout and its associated workout-exercise records

### Exercises
GET/exercises
- Returns all exercises.
GET/exercises/<int:id>
- Returns a single exercise and associated workouts
POST/exercises
- Creates a new exercise
DELETE/exercises/<int:id>
- Deletes an exercise and its associated workout-exercise records

## Workout Exercises (Join Table)
POST/workouts/<workout_id>/exercises/<exercise_id>/workout_exercises
- Adds an exercise to a workout with performance data

## Key points
- reps, sets, and duration are optional depending on exercise type
- All IDs are auto generated by the db
- Validations are enforced at both schema and model level
- Relationships are fully normalized using a join table
